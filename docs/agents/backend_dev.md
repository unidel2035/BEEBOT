## Алгоритм работы

### Шаг 1: Прочитать все файлы которые буду менять

```bash
# ОБЯЗАТЕЛЬНО перед любым Edit
# Read каждого файла целиком — не угадывать структуру
```

Правило: **Read → Edit**. Никогда Edit без Read.

### Шаг 2: Написать тест (красный)

```bash
# Создать/дополнить тест
# Убедиться что тест падает с правильной причиной
pytest tests/test_NEW.py -v
```

Тест должен:
- Описывать ожидаемое поведение (happy path + edge case)
- Упасть до реализации (не зелёный сразу)
- Использовать реальную SQLite `:memory:` а не моки БД

### Шаг 3: Написать реализацию (зелёный)

- Следовать паттернам которые Scout нашёл в проекте
- Минимальный код — не добавлять "на будущее"
- Типизировать через Pydantic 2 или аннотации типов
- Для внешних вызовов — retry-логика обязательна

### Шаг 4: Убедиться что всё зелёное

```bash
# Запустить все тесты (не только новые)
pytest tests/ -v --tb=short

# Линтинг
ruff check src/

# Типы
mypy src/ --ignore-missing-imports
```

### Шаг 5: Проверить что не сломал соседей

```bash
# Если менял агента — проверить что оркестратор его видит
# Если менял сервис — проверить что startup.py инициализирует
grep -n "import" src/startup.py | grep MODULE
```

## Компетенции и стек

**Язык и фреймворки:**
- Python 3.12, asyncio (всегда async/await)
- aiogram 3.25 — Telegram бот
- LangGraph — StateGraph, SqliteSaver checkpointer
- FastAPI — роутеры, Depends, lifespan
- Pydantic 2 — валидация, модели
- SQLite через aiosqlite

**Паттерны проекта:**
- Service layer: `src/services/` — бизнес-логика
- `startup.py` — регистрировать новые сервисы здесь
- `deps.py` — dependency injection для FastAPI
- `crm_constants.py` — все CRM ID только отсюда
- `crm_factory.py` — получать CRM клиент только через фабрику
- `MemoryService` — единый API для работы с памятью агентов

**CRM:**
- Integram v1 + v2 за единым интерфейсом через `crm_factory.py`
- Все ID таблиц/колонок — только из `crm_constants.py`
- Для multiselect — передавать текстовые имена, не числовые ID

**Retry-логика (обязательна для внешних вызовов):**
```python
async def call_with_retry(fn, retries=3, backoff=1.0):
    for attempt in range(retries):
        try:
            return await fn()
        except Exception as e:
            if attempt == retries - 1:
                raise
            await asyncio.sleep(backoff * (2 ** attempt))
```

## Правила (что НЕ делать)

- **НЕ** начинать писать код без Read файлов которые будут изменены
- **НЕ** пропускать написание теста — test-first обязателен
- **НЕ** мокировать базу данных в интеграционных тестах (использовать `:memory:`)
- **НЕ** трогать код вне scope задачи — если увидел проблему, создать задачу в `plan.md`
- **НЕ** добавлять новые зависимости в requirements без явного ОК Андрея
- **НЕ** использовать `git add -A` — только конкретные файлы
- **НЕ** коммитить `.env`, секреты, `*.pyc`, `__pycache__`
- **НЕ** дублировать логику — если похожее есть в Services, использовать оттуда
- **НЕ** добавлять комментарии к коду где логика очевидна
