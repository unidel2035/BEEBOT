## Фаза 3.1: Ассистент пчеловода ✅ (23.03.2026)

Реализован `AdminChatAgent` — LLM-ассистент для ADMIN_CHAT_ID с CRM-снимком:
- `/admin` включает режим личного ассистента
- Снимок CRM: заказы + помесячная статистика + позиции + топ товаров + клиенты + склад
- История диалога в памяти (последние 20 сообщений)
- Использует Groq (llama-3.3-70b), готов к переключению на Anthropic API

**Задачи:**
- [ ] Создать единую функцию `update_status_and_notify(order_id, new_status, source)` в `src/notifications.py`
- [ ] Подключить её в трёх точках: `admin.py` (/status команда), `web/api.py` (PATCH /status), `tracker.py` (авто)
- [ ] Удалить дублирующую логику из `src/web/notifications.py` (оставить только TG-функции)
- [ ] Тест: смена через TG-бот → проверить SSE + уведомление пчеловоду

### 3.3 Исправить порт в tracker.py (P1 — быстро)

**Файл:** `src/delivery/tracker.py:26`

### 3.4 Долгосрочная память через Integram (P2)

**Идея:** Разговоры пользователей сохраняются в Integram и используются в следующих сессиях.

**Задачи:**
- [ ] Создать таблицу «Диалоги» в bibot: user_id, дата, тема, ключевые факты, теги
- [ ] Создать таблицу «Профили пользователей»: предпочтения, здоровье, история продуктов
- [ ] Перед каждым ответом консультанта: поиск истории → обогащение контекста
- [ ] После ответа: сохранить ключевые факты из диалога
- [ ] Команда `/remember <факт>` для пчеловода — добавить знание в базу
- [ ] Автообновление профиля: если пользователь упомянул болезнь/интерес — сохранить

**Результат:** Бот помнит что пользователь интересовался пергой при язве желудка → учитывает при следующем обращении.

Разбить монолитный `src/web/api.py` (1 384 строки) на 6 модулей:

```
src/web/
├── api.py           → main router + startup + общие зависимости
├── api_auth.py      → /api/login, /api/users
├── api_orders.py    → /api/orders/*
├── api_clients.py   → /api/clients/*
├── api_products.py  → /api/products/*
├── api_dashboard.py → /api/dashboard/*
└── api_sse.py       → SSE events
```

**Идея:** BEEBOT становится агентом в AgentBus (dronedoc2026/backend, порт 8081).

**Задачи:**
- [ ] При старте бота: POST `/api/agent-bus/register` → `{agentId: "beebot", capabilities: ["crm", "kb-search", "orders"]}`
- [ ] Обработчик входящих сообщений: `/api/agent-bus/inbox/beebot`
- [ ] Фоновая задача: heartbeat каждые 30 сек
- [ ] API-инструменты для других агентов: поиск по KB, получение статуса заказа
