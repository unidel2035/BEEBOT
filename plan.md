## Фаза 8: Качество и стабильность (P1)

### 8.1 Разбить bot.py на Router-модули

**Проблема:** `src/bot.py` 1 899 строк — хэндлеры всех ролей в одном файле.

**Решение:** aiogram Router архитектура:
```
src/routers/
├── user.py         # /start, /help, /products, /ask, /voice, /cancel
├── worker.py       # worker:queue, worker:order:*, worker:take:*, worker:done:*
├── admin.py        # /admin, /stats, /faq, /yt_check, /yt_update
└── fsm_order.py    # OrderFSM шаги + InspectFSM шаги
```
- [ ] Создать src/routers/ с 4 файлами
- [ ] Перенести хэндлеры из bot.py
- [ ] Обновить bot.py — только startup/shutdown + dp.include_router()

### 8.2 Убрать дублирование в AdminChatAgent

**Проблема:** `_get_crm_context()` и `_build_context_from_snapshot()` — почти идентичный код ~330 строк.

**Решение:** единый метод `_build_context(orders, products, clients)`:
- [ ] Выделить `_format_context(orders, products, clients) → str`
- [ ] `_get_crm_context()` → загружает данные → вызывает `_format_context`
- [ ] `_build_context_from_snapshot()` → достаёт из snapshot → вызывает `_format_context`

### 8.3 Добавить `inspect` в оркестратор

**Проблема:** InspectorAgent не достижим через оркестратор — только через /inspect.

- [ ] Добавить `inspect` в `_INTENT_SYSTEM` (orchestrator.py)
- [ ] Добавить `inspect` в `_fast_classify` (ключевые фразы: «осмотр улья», «диагностика», «осмотри»)
- [ ] Добавить ноду `inspector` в LangGraph граф
- [ ] Хэндлер: passthrough → bot.py запускает InspectFSM

### 8.4 UDS: сопоставление товаров по SKU

**Проблема:** `_build_order_items()` ищет по имени → `product_id=0` для многих товаров.

- [ ] В `integram_client` добавить `get_product_by_sku(sku_uds: str) → Product | None`
- [ ] В `_build_order_items()`: сначала искать по `good["sku"]` (externalId), fallback — по имени
- [ ] Синхронизировать `sku_uds` в Integram для всех 76 товаров

## Фаза 9: Расширение функциональности (P2)

### 9.1 Кнопки-меню в боте

**Двухуровневое меню:** постоянная нижняя панель (ReplyKeyboard) + inline-подменю.

**Панель пользователя:**
```
💬 Спросить  │ 📦 Продукты │ 🛒 Заказать
🔍 Осмотр    │ 🎙 Голос ▾  │ ❓ Помощь
```
**Панель администратора (дополнительно):**
```
📊 Статистика │ 🤖 Ассистент │ 🛠 Dev-задача
```

- [ ] `_build_main_keyboard(role)` — ReplyKeyboardMarkup
- [ ] Хэндлеры текстовых кнопок → redirect на команды
- [ ] «🎙 Голос ▾» → существующее inline-меню /voice

### 9.2 PDF-отчёты и расширенная аналитика

- [ ] Экспорт PDF-отчётов: выручка за период, топ товаров, ABC-анализ клиентов
- [ ] Прогноз спроса на следующий месяц через LLM + статистику
- [ ] Алерты при низком складском остатке (<5 шт.) → авто-уведомление пчеловоду в Telegram
- [ ] Сравнение периодов: этот месяц vs прошлый, этот год vs прошлый

### 9.3 Советы пчеловода (Integram → BEEBOT)

Таблица `DEV_ADVICE` уже создана. Интегрировать в консультанта:
- [ ] При старте загружать советы категории `клиент` из Integram

- [ ] Команда `/advice <текст> [категория]` — добавить совет через бот

## Фаза 10: Инфраструктура и надёжность (P3)

### 10.1 Устранение SPOF — hive

Hive — единая точка отказа для Groq + Telegram + DEVBOT.

**Вариант A — Перенос на VPS (при апгрейде RAM до 4 GB):**
- [ ] Groq-прокси как контейнер на VPS
- [ ] Telegram SOCKS5 на VPS (если IP разблокирован)
- [ ] DEVBOT на VPS с доступом к Claude CLI

**Вариант B — Резервный туннель:**
- [ ] Мониторинг доступности hive-туннеля
- [ ] Fallback-цепочка: Groq → Groq с резервным IP → упрощённый ответ без LLM
- [ ] Алерт пчеловоду при потере туннеля

### 10.2 SSL + домен для веб-панели

- [ ] Nginx reverse proxy с Let's Encrypt SSL
- [ ] Домен (например `beebot.dmitrovykh.ru`) → https://
- [ ] HSTS, CSP-заголовки
- [ ] JWT в httpOnly cookie (устранить XSS-риск)

### 10.3 Платёжная интеграция

- [ ] YooKassa или Тинькофф — онлайн-оплата заказов в Telegram
- [ ] QR-код для оплаты (Telegram Stars или СБП)
- [ ] Авто-смена статуса после подтверждения оплаты
- [ ] Webhook для уведомлений от платёжного шлюза

### 10.4 Резервное копирование

- [ ] Ежедневный бэкап `data/memory.db` (SQLite) на Yandex Disk / S3
- [ ] Экспорт данных CRM раз в неделю (CSV через API)
- [ ] Backup FAQ-queries.json

### 10.5 Апгрейд VPS

Текущий RAM: 1.921 GiB (BEEBOT использует ~838 MiB).

- [ ] Апгрейд до 4 GB RAM при добавлении новых контейнеров
- [ ] Возможность перенести Groq-прокси на VPS
