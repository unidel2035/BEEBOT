## Фаза 3: Расширение — Интеллект и Память

**Цель:** Превратить бота из инструмента ответов в живого ассистента, который учится из общения и запоминает пользователей.

### 3.1 Диалог пчеловода с LLM-ассистентом (P1) — ✅ выполнено 23.03.2026

**Идея:** Пчеловод (ADMIN_CHAT_ID) общается с Claude напрямую через Telegram — бот становится персональным ассистентом.

**Задачи:**
- [ ] Добавить `anthropic` в requirements.txt + `ANTHROPIC_API_KEY` в .env
- [ ] В обработчике сообщений: если `from_user.id == ADMIN_CHAT_ID` → Claude API вместо Groq
- [ ] Системный промпт: контекст проекта, CRM-данные, роль ассистента
- [ ] Инструменты для Claude: получить статистику заказов, список клиентов, остатки склада
- [ ] Команда `/claude` — явно переключиться в режим Claude (для группы)
- [ ] История диалога сохраняется в Integram (таблица «Диалоги»)

**Результат:** Пчеловод пишет в Telegram: «Что заказывали больше всего в марте?» — получает ответ с данными из CRM.

### 3.2 Долгосрочная память через Integram (P1)

**Идея:** Каждый разговор пользователя с ботом сохраняется в Integram и используется в следующих сессиях.

**Задачи:**
- [ ] Создать таблицу «Диалоги» в bibot: user_id, дата, тема, ключевые факты, теги
- [ ] Создать таблицу «Профили пользователей»: предпочтения, состояние здоровья, история продуктов
- [ ] Перед каждым ответом консультанта: поиск истории пользователя → обогащение контекста
- [ ] После ответа: сохранить ключевые факты из диалога
- [ ] Команда `/remember <факт>` для пчеловода — добавить знание в базу
- [ ] Автоматическое обновление профиля: если пользователь упомянул болезнь/интерес — сохранить

**Результат:** Бот помнит что пользователь интересовался пергой при язве желудка → при следующем обращении учитывает это.

### 3.3 Обучение через общение (P2)

**Идея:** Пчеловод «обучает» бота, рассказывая факты в Telegram.

**Задачи:**
- [ ] Команда `/teach <текст>` — добавить факт в базу знаний (как новый чанк)
- [ ] Парсинг обучающих сообщений → добавление в FAISS-индекс без пересборки
- [ ] Версионирование KB: фиксировать дату добавления каждого чанка
- [ ] Дашборд «Что нового в KB» — последние добавленные знания

### 3.4 Исправление синхронизации уведомлений (P1) — ✅ частично выполнено 23.03.2026

Подробнее в [analysis.md #3.2](analysis.md).

**Выполнено:**

- [x] `notifications.py` использует `TG_SOCKS_PROXY` через `_make_client()`

**Остаётся (P2):**

### 3.5 Реальная пагинация на фронтенде (P2)

## Фаза 4: Экосистема — Подключение к AgentBus

**Цель:** Интегрировать BEEBOT в единую экосистему агентов (DronDoc, Buddy, TraderAgent) через AgentBus.

### 4.1 Регистрация в AgentBus (P2)

**Идея:** BEEBOT становится агентом в AgentBus (dronedoc2026/backend, порт 8081).

**Задачи:**
- [ ] При старте бота: POST `/api/agent-bus/register` → `{agentId: "beebot", capabilities: ["crm", "kb-search", "orders"]}`
- [ ] Обработчик входящих сообщений: `/api/agent-bus/inbox/beebot`
- [ ] Фоновая задача: heartbeat каждые 30 сек
- [ ] API-инструменты для других агентов: поиск по KB, получение статуса заказа, создание заказа

### 4.2 Общий профиль пользователя (P2)

**Идея:** Профили клиентов из BEEBOT доступны другим ботам (Buddy, DronDoc).

**Задачи:**
- [ ] Использовать `UserProfileService` из Buddy для хранения профилей
- [ ] При создании клиента в BEEBOT → создать/обновить профиль в shared Integram (база `fst`)
- [ ] При консультации: поиск профиля → персонализированный ответ

### 4.3 Перенос LLM-инфраструктуры с hive на VPS (P3)

**Задачи:**
- [ ] Оценить возможность запуска Ollama на VPS (4 GB RAM minimum)
- [ ] Fallback-цепочка: Groq API → Ollama на VPS → упрощённый ответ
- [ ] Мониторинг доступности hive-туннеля с авто-fallback
